一、题目大意

本题输入为一串长度为$n$的字符串，表示一个含有加法、减法、乘法、乘方以及括号运算的多项式$f(x)$。我们要做的是将多项式展开为如下的标准形式：

并按照次数从高到低的顺序输出每一项的系数$a_i$对1000000007取模的值。

题目数据保证$0 < n \leq 1000000$，计算过程中涉及的乘方运算最多不超过$4$次乘方，涉及的多项式的最高次数不超过$64$。

二、算法分析

我们首先考虑不含字母$x$，只有数字的表达式的求值问题，容易想到课上所讲的中缀表达式求值的方法。于是可以类推到本题。在本题中，我们使用栈1存储所有需要操作的多项式，称为操作数栈；使用栈2存储字符串中的运算符和左右括号，称为操作符栈。这样我们就可以将中缀表达式求值的算法推广到对多项式的化简问题中。

结构体中定义了一个整型变量max_deg，表示存储的多项式的最高次数。由于计算过程中涉及的多项式最高次数不超过64，则定义一个65位的整型数组，其中第$i$位存储$x^i$项的系数。接着为了方便进行多项式之间的运算，对+、-、*、^四个运算符进行了结构体内重载。

在对加减号的重载中，我们令结果的最高次数为两个被操作多项式的最高次数中较大者，即接着对两个num数组进行按位加或减即可得到结果多项式的num数组。

在对乘号的重载中，我们令结果的最高次数为两个被操作多项式的最高次数相加，然后在计算系数的时候模拟竖式运算即可。

在对乘方的重载中，我们额外定义一个多项式结构体temp来存储原有的底数多项式，定义ans用来保存最后的答案，然后获取指数，调用已经重载完成的*运算符即可。


所有的重载过程中，都需要适当处理模数问题。

若遇到$x$，则定义一个最高次数为1，num数组只有num[1] = 1的多项式，进入操作数栈。

若遇到操作符，则比较当前操作符与栈顶操作符的优先级。若优先级为<，则推迟计算，当前操作符入栈；若优先级为>，则连续两次取出操作数栈中的栈顶元素，根据操作符栈顶运算符进行计算，结果进入操作数栈；若优先级为=，则说明有一对括号匹配成功，此时操作符栈栈顶为左括号，则将左括号弹出，括号内计算结束。当扫描到最后一位，处理结束，则操作数栈栈顶元素即为整理完毕的多项式，并作为calculate()函数的返回值。

在最终输出的时候，注意按照从高位到地位的顺序输出即可。

四、实现中值得注意的细节

  1. 在重载运算符时，需要注意出现最高次项对应的系数为0的情况，例如计算过程中出现了x-x这样的运算。于是我们额外定义一个名为process的成员函数，每当出现num[max_deg] = 0的情况时，则执行max_deg --，直到num[max_deg] != 0或者max_deg = 0。

  2. 在重载运算符时，为了节省时间，循环的终止数设置为多项式的max_deg，而不是每次循环都要遍历数组的65位。例如在运算$x+x$时，直接循环两次即可，相比循环65次，在运算次数很多时，可以大量节省时间。

  3. 在涉及到数字之间的乘法的操作（重载*运算符以及处理字符串遇到不止一位的数字）时，由于运算中的结果可能超过int可能表示的范围，则采用如下的方式（乘以1ll）将结果转为long long类型。
  4. 在初次提交时，由于题中设定输入字符串的长度不超过1000000，于是我设置const int N = 1000005，将操作数栈和操作符栈中的数据域大小都设置为N，提交之后造成了Memory Limit Error。经过分析，我发现操作数栈不需要申请这么多空间。事实上，在最劣的情况下，每三位字符中都至少有一个操作符，例如以下的字符串2x+2x+2x+2x+...，此时2，x均为操作数，+为操作符。其他情况下，数字位数变多，则必然操作数总数会减少。则操作数的总数最多不会超过2 * N / 3，于是设置const int N2 = 2 * N / 3，将操作数栈中数据域的大小设置为N2，解决了内存超限的问题。
  5. 在给输入字符串补充乘号的时候，没有采用$O(n)$的insert()等方法，而是采用了新开字符串，并且依次append()的方法，将每次补乘号的时间缩短到了$O(1)$。

五、时间与空间复杂度分析

  1. 时间复杂度分析：在本题中，对字符串的操作的复杂度整体是$O(n)$的，而在运算数运算过程中，单次运算最劣复杂度为$O(m^2)$，则本题算法理论时间复杂度为$O(m^2 n)$。
  2. 空间复杂度分析：在本题中，主要的空间开销是在定义操作符栈和操作数栈的过程，则本题算法理论空间复杂度为$O(mn)$。
