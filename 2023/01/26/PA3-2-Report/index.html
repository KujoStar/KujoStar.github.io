PA3-2 Report

Created2023-01-26|Updated2023-01-26|学习历程

一、题目大意

给定一个长度为$n$的$01$串$A$，寻找最短的不是$A$的子串的$01$串$B$并输出，如有多个则输出字典序最小的。

其中$1 \leq n \leq 16777216$。

二、算法分析

本题中，由于空间限制为$6$MB，若采用string类型输入字符串，最大需要$\displaystyle \frac{16777216}{1024 \times 1024}=16$MB的空间进行存储，显然会造成空间超限的问题。于是考虑更节省空间的存储方式——位图。使用位图存储方式，由于每个char类型变量中都有$8$个bit，则可以将存储空间缩小为原来的$\frac{1}{8}$，足以通过本题。

下面是如何寻找答案的问题。设子串长度为$x$，则不同的$01$串一共有$2^x$种，而字符串中最多有$n-x+1$个长度为$x$的子串。因此只要子串长度$x$满足$2^x>n-x+1$恒成立，则一定可以找到答案。注意到以下这个显然的事实：

而当$x=24,n=2^{24}$时，上述不等式成立，则答案字符串的长度$l$一定满足$1\leq l \leq 24$。因此我们可以直接枚举这$24$个长度，一一检查即可。

对$1 \leq l \leq 24$的每个长度$l$，输入的字符串已经被存储在位图s中，我们可以构建一个空位图t，之后遍历位图s中所有长度为$l$的子串，设每个$01$子串对应的十进制表示为$k$，将位图t中的第$k$位赋值$1$。遍历完成后，寻找位图t中第一个值为$0$的位，若能找到，位数的二进制表示（需要补零达到相应长度）的字符串便是答案。

四、实现中值得注意的细节

  1. 要熟记位运算符与普通四则运算符之间的优先级关系，避免在实现时出现运算错误。
  2. 最开始，我没有将两个位图s和t定义为全局变量，而是在main()函数中定义，这导致了栈空间分配不足，终端调试时甚至无法输入数据。因此还是尽量定义全局变量为好。

五、时间及空间复杂度分析

  1. 本题中，对$1 \leq l \leq 24$的每个长度$l$，分别对位图s和t进行了遍历，则时间复杂度为$O(n)$（常数略大）。
  2. 本题中，主要在构造位图时申请了空间，空间复杂度为$\displaystyle O\left(\frac{n}{4} \right)$。
