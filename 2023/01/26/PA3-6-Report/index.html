PA3-6 Report

Created2023-01-26|Updated2023-01-26|学习历程
|Post View:

一、题目大意

本题中，给定$n$个$d$维向量$\mathbf{x}_1,\cdots,\mathbf{x}_n$，$q$次询问，每次寻味给出一个$d$维向量$\mathbf{x}$，求向量$\mathbf{x}$与给出的$n$个向量距离的平方的最小值。
数据满足$2 \leq d \leq 5$，$1 \leq n \leq 10^5$，$1 \leq q \leq 2 \times 10^5$，所有向量每个分量的绝对值不超过$10^7$。

二、算法分析

由题意可知，本题是一个高维的最近邻查询问题。为高效处理多维空间的数据信息，因此考虑采用kd树进行存储与查询。

kd树建树的核心思想为，每次选择一个维度中的一个基准点之后按照这一维度上值的大小关系将这$n$个点构成的$k$维空间内的超长方体进行分割，之后向两侧的超长方体递归，从而可以建立kd树。为了尽量达到平均分割，我们选择所有点在这一维度上的分量值得中位数作为基准，因此我们需要获取$n$个值的中位数，也即将第$k$大的值置于序列第$k$位的kth_element算法。

四、实现中值得注意的细节

  1. 根据本题的数据范围，最终的结果（距离的平方）可能超过int数据范围，因此相关变量以及函数返回值需要使用long long数据类型进行存储。
  2. 在输入给定的n个向量时，可以令数组下标从$1$开始进行存储，这样子可以在point数组中预留出首位用于处理特殊情况。例如上述的query函数中if(x == 0) return;一行就是利用的这一便利。

五、时间与空间复杂度分析

  1. 在本题中，主要的空间使用是用于存储给出的$n$个$d$维向量的每一维的数据信息，则空间复杂度为$O(dn)$。
  2. 在本题中，实现kth_element算法的时间复杂度为$O(n)$，建树时每次根据中位数划分，则建立kd树的时间复杂度为$O(n\log{n})$。每次查询的时间复杂度为$O(\log{n})$，则总时间复杂度为$O((n+q)\log{n})$。
